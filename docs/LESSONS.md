## Process / working model
- **Delegate volume, verify the consequential claims yourself.** Background sub-agents carried real load well (`sonnet` for verify-heavy research, `opus` for taste-heavy docs). But the tech lead re-ran the single most verdict-affecting agent finding by hand (the integer-overflow default) instead of trusting the summary â€” and it mattered. Independently re-verify anything that changes a verdict. â†’ [CLAUDE.md](../CLAUDE.md) Delegation policy.
- **Cold-test the handoff.** A fresh-context, read-only agent given only the repo + "proceed" is a cheap, honest test of whether the docs actually carry the next session. It caught an uncommitted-instruction gap we'd otherwise have shipped. â†’ [SESSION_WRAP.md](SESSION_WRAP.md) Â§5.
- **Two entry points, one destination.** A session started in `~/repos/mettle` auto-loads the repo `CLAUDE.md`; one started in the website dir auto-loads the cross-session memory instead. Both must funnel to `docs/STATE.md`. Keep both current.
- **Never let project-owned code live in a git-ignored directory.** mt-006's delegated shim landed in `oracle/` (ignored wholesale for legal hygiene) â€” it built and tested fine locally while being silently uncommittable; a fresh clone would have lost it. Review caught it; the fix is structural: first-party code goes inside a crate (`crates/als-conform/shim/`), ignored dirs hold only re-downloadable third-party artifacts. Check `git status` untracked output against "what did we write ourselves" at review time.
- **Byte-identical reproduction audits your documentation for free.** mt-009's acceptance bar ("the fetch script must rebuild `corpus/` byte-for-byte") surfaced that the provenance manifest under-documented the upstream pipeline (a 4th patch the prose called 3). Prose records drift; a reproducibility check against reality is the cheapest honest reviewer. Prefer "regenerate and diff" acceptance criteria over "matches the description" wherever possible.
- **Spec from the implementation's sources, then jar-probe the surprises â€” never from the published grammar or memory.** Rung-1's syntax contract came from reading `Alloy.lex`/`Alloy.cup`/`CompFilter` at the jar's build commit; that surfaced behavior no documentation states (five token-stream rewrites, `::` â‰ˇ `.`, `Version.experimental=true` gating live syntax, hex-digit *pairs*). Where the generated-lexer semantics (longest-match + rule-order tie-breaks) made the raw patterns misleading, one round of jar probes settled it (`1_000` is illegal!). The spec doc records *behavior*, with every surprising claim jar-verified and dated. â†’ [reference/alloy6-grammar.md](reference/alloy6-grammar.md).
- **A delegate's "benign divergence" list is a review queue, not a footnote.** mt-011's report flagged four divergences as benign; adjudicating each against the reference showed two were not (`;`-grouping changed formula semantics under `after`; prefix over-acceptance broke error parity). The delegate's *report* was the review input that made this cheap â€” require divergence lists in every delegation report, then verify each one independently before merge.
- **A grammar-narrowing change costs double: parser AND printer must move in lockstep.** mt-014's binder-composition fix (restricting what parses) immediately broke the parseâ†’printâ†’reparse round-trip â€” the printer's "when do parens matter" logic embodied the old, looser grammar and started under-parenthesizing. The structural cure was the mt-012 pattern extended: put the rule itself (`child_binder_budget`) in the shared `prec` module both sides consume, so drift is impossible by construction. Budget any future grammar-restriction bead as a parser+printer change, never parser-only, and let the round-trip corpus test be the tripwire (it caught this one).
- **Big corpora + real-world junk find what specs and unit tests don't.** The alloy4fun differential pass (mt-013) and the mutation fuzzer (mt-014) each found real bugs the 167-file curated corpus never touched: student typos exercised grammar corners (bare `disj`, trailing `^`), byte-level mutants found the printer under-parenthesization, and deep-nesting stressors proved the parser could SIGABRT a debug test worker at ~212 paren levels (fixed with `MAX_EXPR_DEPTH`, a *measured* limit â€” measure stack headroom on the smallest relevant stack, don't guess). Cheap, deterministic, seeded â€” keep both in CI forever.
- **Concurrent agents in one working tree work if partitioned by path.** Two background delegations + tech-lead coding ran simultaneously with zero conflicts by (a) giving each a disjoint file-path scope up front, (b) banning workspace-wide `cargo fmt`/`clippy` in agent specs (package-targeted `-p` only), and (c) the tech lead committing per-scope with pathspec `git add`. Cheaper than worktrees for disjoint work; use worktrees only when paths must overlap.
